The Tigerseye Plug-in Extensions

Different extensions are used to support a modular and extensible architecture. Important extensions are used to introduce new embedded DSLs (EDSLs).

Offered by Builder (de.tud.stg.popart.builder)

LanguageProvider

ID de.tud.stg.popart.builder.languageProvider

At least one plug-in must implement this extension point. The language provider is responsible to load the classes that describe the languages. (currently extension used for that is popartLanguages)

Transformer Selector

ID de.tud.stg.popart.builder.transformerSelector

Unknown purpose (Not used). Original intent might have been to make a selection for each DSL of necessary transformers accessible. If that's the case The logo DSL could for example define to need a Translation transformer if it is used together with a different set of keywords.

Transformers

ID de.tud.stg.popart.builder.transformers

A transform supports functionality for transformation. For example the Translation transformer provides the ability to use a different set of keywords if a translation file is provided describing the necessary mapping of the used to the original keywords.

Offered by Eclipse

Natures

Used to describe the kind of project. When extending a nature it is possible for a plug-in to configure the project s.t. it has access to necessary plug-ins and output folders. The Tigerseye plug-in requires the Groovy nature. It should be possible to add both at the same time. Hitherto they have to be added separately. When the groovy nature is not available the builder might still be functional but the launcher won't be able to launch a file since the launch process is merely a delegation to the groovy launcher to launch the DSL-to-Groovy-translated files.

Natures define, if necessary, their incremental builders. In case of tigerseye a specific builder is necessary defined through the builders extension point.

The popart plug-in had to natures PopartNature and DSLBuilderNature. It should be sufficient now to have one tigerseyeNature
